Class Overview

TpAnimation is the main class responsible for playback and managing the animation of Processing sketch's fields.
It allows for multiple types of playback like single forward, backwards, loop and loop with mirror (back and forth), as well as rendering your animation to frames.

TpAnimation allows you to configure value transitions once, during sketch initialization, in a little declarative way, and then does all the calculations for you behind the scenes, so you can focus on your awesome visuals and stuff, instead coding transitions, timers, interpolations and easings.
You can choose from 32 available easing functions (30 of them come from Robert Penner's easing library.
See TpEasing for the list of all available easing functions waiting for you.
The constructor, setters and add.. methods are created in a way that enables you to call them in a chain like in the example below.
Keep in mind that when the animation is playing, looping or rendering the animated variables' values will be set at the beginning of each draw() call.
The animation won't work if you override the values inside your sketch's draw() method.

When your animation looks perfect you can easily render it as frames with render() method.
You can find the frames in the animationOutput/ directory in your sketch's folder. You have to assemble generated frames to a video on your own with e.g. ffmpeg, PDE's Movie Maker, After Effects or any other tool you like.
Default render frame rate is 30fps, you can change it with setFrameRate() method

Example basic usage:


import com.tajnyprojekt.tpanimation.*;


TpAnimation animation;

// create variables controlling the state of sketch in the main scope
// and mark them as public!
// the initial values will be overridden by the animation
public float a = 170;
public float b = 140;
public float x = 170;

void setup() {

    size(400, 400);

    // create and initialize the animation
    animation = new TpAnimation(this, 1000)
        .setLoopMirror(true)
        .addVariableAnimation("a", 0, 280, TpEasing.ELASTIC_OUT)
        .addVariableAnimation("b", 0, 240, TpEasing.BOUNCE_OUT)
        .addVariableAnimation("x", 170, 230, 800, 1400)
        ;

    // loop and start the animation
    animation.loop();
}


void draw() {

    // use your animated variables to create a simple blinking eye animation
    background(0);
    noStroke();
    fill(69, 247, 148);
    ellipse(200, 200, a, b);
    fill(0);
    ellipse(x, 200, 120, 120);
}

public TpAnimation setLoopMirror (boolean isMirroring)

Sets looping in mirror mode. That means that after calling loop() the animation will play from the beggining to the end, and then in reverse from the end to the beginning and so on.
When mirror mode is disabled, each loop is starting from the beginning of the animation.

public TpAnimation setOutputFilenamePattern (String outputFilenamePattern)

Sets a String pattern used to generate rendered frame's filename with String.format().
Must contain an integer flag %d or it's variation and an extension, read more in Java Formatter docs.
Extension can be any image extensions supported by Processing, read more in Processing save() docs.
Don't start the filename pattern with slash '/'!

Example:
"frame-%04d.jpg" // tells the formatter to pad the frame number with four leading zeros and jpg extension
"%d.png" // default value - results in 0.png, 1.png and so on
Not allowed to call during animation rendering.

Parameters
outputFilenamePattern pattern used to generate rendered frame's filename
